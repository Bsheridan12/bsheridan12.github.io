The Classless Bad Guy

Ruby Classes

11/27/14

In Ruby, everything is an object and every object is an instance of a class. The object's class defines a set of built in attributes and methods that every object within that class shares. Preexisting classes, such as hashes and strings, already have a set of methods for you to use. It is worth noting that every new class can inherit attributes and methods from a parent class but for now we will be creating our own independent class.

Creating The Class

Let's say you're creating a video game and you want to have a lot of generic enemies for your hero to fight along the way. Instead of rewriting or just copy and pasting the code for your bad guys every time you want them to show up, you can create a class. 
 class Bad_guy

    end
Alright, it's a start, but our bad guy is looking a little empty.

Defining Attributes

We can now give our bad guys some characteristic that each new guy will share. 
 def initialize
    @health = 100
    @damage = 5
    @taunt = ["Pathetic", "Not a chance!",
    "Your mother was a goat and your father 
    smelled of elderberries!"]
  end

Creating Methods

Now that we have a basic template that defines our bad guys, we can give them some actions to perform. 
 def attack
    puts "Bad Guy attacked you for #{@damage} damage!"
  end

  def taunt
    puts @taunt.sample
  end

  def flee
    puts "I've made a terrible mistake!"
  end

Creating New Instances

With our bad guy fully equipped with some attributes and actions we can start creating new instances and calling our methods on them! 
 henchman = Bad_guy.new

      henchman.attack
      "Bad GUy attacked you for 5 damage!"

      henchman.tanut
      "Your mother was a goat and your father 
      smelled of elderberries!"

      henchman.flee
      "I've made a terrible mistake!"

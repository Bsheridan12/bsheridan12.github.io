Ruby vs. JavaScript

Iterations

12/11/14

After several weeks of working in Ruby, we have started learning JavaScript. At first I was not at all a fan. Ruby syntax, to me, feels a lot cleaner and is much nicer to read than JavaScript. Now that I have worked with it for a little while I have warmed up to it though. We had a challenge this week that required us to make a pretty basic game and once I was able to move little icons across the screen I decided to forgive JavaScript it's faults.

One thing that I had some trouble with this week was iterating through an array. Here's what iteration looks like in both languages.

Ruby

Ruby has some built in methods that make iteration very simple. 

 
    array = [1, 2, 3, 4, 5]
    array.each { |num| puts num + 1 }

    # 2
    # 3
    # 4
    # 5
    # 6

JavaScript

JavaScript does not have these methods so every time you need to iterate you essentially have to build your own each method using a for loop.

    array = [1, 2, 3, 4, 5]
    for (i = 0; i < array.length; i++) {
      console.log(array[i] + 1)
    }

    // 2
    // 3
    // 4
    // 5
    // 6

A little more work here than in Ruby but still pretty easy to make sense of, syntax aside. First we declare a counter for the for loop, i. Then we tell the for loop only to run as long as our counter is less than the length of our array so that we do not create an infinite loop. Next we increment our counter by 1 so that we hit every element of the array. Finally we pass essentially the same information into the body of the for loop as did in the #each block in Ruby.

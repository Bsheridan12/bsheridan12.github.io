Enumerable#map

Iterate over Collections

11/17/14

In Ruby, everything is an object. If you want to interact with these objects or change them in some you can use methods. You can create your own methods but Ruby also has a huge bank of predefined methods that you can call on as you like. Different objects use different types of methods. Objects that are a collection of items, like an array or a hash, use a specific type of method to preform an action on each individual item within them. These methods are called enumerables. We're going to be looking at how the #map enumerable works. Well start with a simple array: 
  array = [1, 2, 3, 4, 5]
Now lets say we want to add 1 to every item in this array. We can do this using #map. First we will call the #map enumerable method on our array and then pass it a block of code that we want it to preform on our array. 
  array = [1, 2, 3, 4, 5]

  p array.map {|x| x+1} 
    # => [2, 3, 4, 5, 6]

  p array 
    # => [1, 2, 3, 4, 5]
So we can see that #map behaved exactly how we expected it to by adding 1 to each element in the array. However, on the third row when we checked the value of array again, we can see it stayed the same. This is a nice feature of #map. It does not alter the original array but rather clones it and preforms it's block of code on the new clone. If you want to change the original array though, no worries! Just slap an exclamation mark on the end of #map and you've done it. 
  array = [1, 2, 3, 4, 5, 6, 7]

  p array.map! {|x| x+1} 
    # => [2, 3, 4, 5, 6]

  p array 
    # => [2, 3, 4, 5, 6]
So there you go, you can now effect great change within your collections.
